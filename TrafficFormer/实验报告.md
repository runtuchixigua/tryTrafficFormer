# TrafficFormer 项目实验报告

## 一、项目概述

### 1.1 项目背景
TrafficFormer 是一个基于 Transformer 架构的网络流量分类项目，将自然语言处理（NLP）技术应用于网络安全领域。其核心创新在于将网络流量数据转换为类似自然语言的序列数据，然后使用 BERT 等预训练模型进行处理。

### 1.2 技术架构
```
网络流量 (PCAP) → 流量分割 → 特征提取 → 序列化 → BERT 模型 → 分类结果
```

### 1.3 硬件环境
- **设备**: MacBook Air (Apple M4)
- **芯片**: Apple M4 (10核心: 4性能核心 + 6能效核心)
- **内存**: 32 GB
- **GPU**: Apple M4 GPU (10核心), 支持 Metal 3
- **运行环境**: Python 3.10 + PyTorch 2.0.0

## 四、结果分析

### 4.1 问题诊断

#### 问题 1: 模型只预测一个类别
**现象**: 模型将所有样本都预测为 email 类别
**原因**: 
- 数据严重不平衡，email 占 38% 的样本
- 训练轮数不足，模型未充分学习
- 学习率可能需要调整

#### 问题 2: 训练损失不稳定
**现象**: Loss 从 1.507 上升到 1.859 再下降到 1.842
**原因**: 
- 学习率可能过大
- 批次大小较小导致梯度估计不稳定

#### 问题 3: 验证准确率无提升
**现象**: 三个 epoch 验证准确率都是 37.8%
**原因**: 
- 模型陷入局部最优
- 需要更多训练轮次

### 4.2 根本原因分析

```
┌─────────────────────────────────────────────────────────────┐
│                    性能不佳的主要原因                        │
├─────────────────────────────────────────────────────────────┤
│  1. 数据不平衡 (影响最大)                                    │
│     - email: 482 样本 vs aim_chat: 54 样本                  │
│     - 模型倾向于预测多数类                                   │
│                                                             │
│  2. 训练不充分                                              │
│     - 只训练了 3 个 epoch                                   │
│     - BERT 微调通常需要 10-20 个 epoch                      │
│                                                             │
│  3. 数据量有限                                              │
│     - 总共只有 1,270 个样本                                  │
│     - 深度学习模型通常需要更多数据                           │
│                                                             │
│  4. 超参数未优化                                            │
│     - 学习率、批次大小等可能需要调整                         │
└─────────────────────────────────────────────────────────────┘
```

## 五、改进建议

### 5.1 数据层面

#### 5.1.1 解决数据不平衡
```python
# 方法 1: 过采样少数类
from imblearn.over_sampling import RandomOverSampler
ros = RandomOverSampler(random_state=42)
X_resampled, y_resampled = ros.fit_resample(X, y)

# 方法 2: 欠采样多数类
from imblearn.under_sampling import RandomUnderSampler
rus = RandomUnderSampler(random_state=42)
X_resampled, y_resampled = rus.fit_resample(X, y)

# 方法 3: 类别权重
class_weights = {0: 5.0, 1: 1.0, 2: 1.5, 3: 2.5, 4: 2.0}
```

#### 5.1.2 数据增强
```python
# 对网络流量数据进行增强
1. 随机修改 IP 地址和端口
2. 随机修改时间戳
3. 添加噪声
4. 数据包重排序
```

### 5.2 模型层面

#### 5.2.1 调整超参数
```bash
# 推荐配置
python fine-tuning/run_classifier.py \
    --vocab_path models/encryptd_vocab.txt \
    --train_path dataset/train_dataset.tsv \
    --dev_path dataset/valid_dataset.tsv \
    --test_path dataset/test_dataset.tsv \
    --pretrained_model_path ../nomoe_bertflow_pre-trained_model.bin-120000 \
    --output_model_path models/finetuned_model_v2.bin \
    --epochs_num 20 \           # 增加训练轮数
    --earlystop 5 \             # 增加早停耐心
    --batch_size 16 \           # 增大批次
    --learning_rate 2e-5 \      # 降低学习率
    --seq_length 320 \
    --config_path models/bert/base_config.json
```

#### 5.2.2 使用更小的模型
```bash
# 如果内存不足，可以使用更小的模型
--config_path models/bert/small_config.json  # 或 mini_config.json
```

### 5.3 训练策略

#### 5.3.1 学习率调度
```python
# 使用 warmup + 线性衰减
--scheduler linear \
--warmup 0.1  # 前 10% 步数用于 warmup
```

#### 5.3.2 梯度累积
```python
# 如果 GPU 内存不足，使用梯度累积
--batch_size 8 \
--accumulation_steps 4  # 等效 batch_size = 32
```
